# Dodo Pizza Project (Backend) 🍕

Прогрессивный бэкенд-сервис (REST API) для автоматизации пиццерии и управления доставкой еды. Проект спроектирован с учетом стандартов промышленной enterprise-разработки, высокой связанности слоев и строгой валидации данных.

---

## 🛠 Технологический стек

* **Язык и база:** Java 17 / Spring Boot 3 / Spring Data JPA
* **СУБД:** PostgreSQL (в изолированном Docker-контейнере)
* **Миграции:** Liquibase (версионирование схем данных, отказ от `ddl-auto: update`)
* **Маппинг:** MapStruct (высокопроизводительное разделение слоев Entity/DTO)
* **Инфраструктура:** Docker, Docker Compose
* **Тестирование:** JUnit 5, Mockito (модульное тестирование сервисного слоя)
* **Документация:** Swagger UI / OpenAPI 3
* **Валидация и логирование:** Jakarta Validation, SLF4J + Logback

---

## 🚀 Ключевые архитектурные решения и бизнес-логика

Вместо классического «принеси-подай» CRUD, в проект заложены реальные бизнес-сценарии:

* **Умный модуль заказов (`OrderService`):** Связь **Many-to-Many** между заказами и продуктами через промежуточную таблицу. Автоматический динамический расчет стоимости доставки на бэкенде (через безопасный `BigDecimal`) на основе пороговой суммы заказа.
* **Атомарность операций (`@Transactional`):** Все сквозные операции сохранения данных защищены каскадными транзакциями. При ошибках на любом этапе происходит гарантированный Rollback.
* **Безопасность данных (Soft Delete):** Реализован механизм «мягкого удаления» продуктов через флаг `active`. Данные не стираются физически, сохраняя целостность исторической аналитики заказов.
* **Защита на подлёте (fail-fast):** Архитектура разделена на Entity и DTO. Входящие JSON-данные проходят строгую проверку через `@NotBlank` и `@NotEmpty`, минимизируя нагрузку на БД.
* **Централизованный Траблшутинг:** Ошибки валидации и кастомные исключения бизнеса (`InvalidProductPriceException`) обрабатываются через `GlobalExceptionHandler`, возвращая клиенту понятный унифицированный `ErrorResponseDTO`.

## 🐳 Быстрый запуск (Docker)

Проект полностью контейнеризирован. Вам не нужно устанавливать локальный PostgreSQL — всё поднимется автоматически.

1. **Клонируйте репозиторий:**
   ```bash
   git clone [https://github.com/your-username/Dodo_Pizza_Project.git](https://github.com/your-username/Dodo_Pizza_Project.git)
   cd Dodo_Pizza_Project

Как запустить:  
1.Клонировать репозиторий.

2. Прописать docker compose up --build.

3. Проверить API через Swagger по адресу http://localhost:8080/swagger-ui.html
